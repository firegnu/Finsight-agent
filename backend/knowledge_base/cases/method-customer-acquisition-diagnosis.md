# 获客下滑根因分析框架（四象限法）

## 何时使用

获客量月环比下滑 > 15%，或连续 2 月低于历史均值 2σ，触发本流程。

## 四象限定位法

按"**获客量 × 激活率**"组合判断根因：

| 获客量 | 激活率 | 最可能根因 |
|---|---|---|
| **↓ 跌** | **→ 稳定** | **渠道问题**：流量不够但进来的客户质量 OK |
| **↓ 跌** | **↓ 同步跌** | **渠道质量降级**：备胎渠道拉来低意愿客户 |
| **→ 稳定** | **↓ 跌** | **产品/定价问题**：看得到但不愿开卡 |
| **↓ 跌** | **↑ 反升** | **主动筛选收紧**：审批门槛提高了（有意的） |

## 调查四象限

### 象限 1：渠道（流量层）

- **渠道构成分析**：TOP 5 渠道月度贡献，有没有单渠道骤降 > 30%？
- **流量异常原因**：平台政策、合作分成、竞品挤压、banner 位降级
- **合作方谈判状态**：有没有续约僵局、纠纷？
- **验证动作**：分渠道看获客，集中度 > 50% 的要特别警惕

### 象限 2：产品（供给层）

- **核心权益**：最近 3 个月有变更吗（积分兑换、年费、透支额度策略）？
- **目标客群匹配度**：本产品的定位客群，渠道拉来的用户是不是这个画像？
- **申请页面**：有没有前端 bug（表单提交率骤降）？
- **验证动作**：申请转化漏斗分析，每一步留存率 vs 历史

### 象限 3：定价（经济层）

- **年费策略**：减免条件是否调整？
- **利率调整**：分期、取现利率变化？
- **竞品对标**：同一客群下，竞品最近推了什么优惠？
- **验证动作**：价格敏感度测试（A/B 测试不同定价档位）

### 象限 4：审批（入口层）

- **审批通过率**：是否自动紧了？（有时风控模型更新会隐性收紧）
- **风控白名单/黑名单更新**：是否把部分客群误伤？
- **信用分数线**：有无调整？
- **验证动作**：审批通过率趋势 + 拒绝原因分布

## 处方对应

| 确诊象限 | 标准动作 |
|---|---|
| 渠道 | 续约谈判 / 多元化布局 / 新渠道拓展 |
| 产品 | 权益升级 / 申请页优化 / 客群定位重审 |
| 定价 | 短期促销 / 长期定价重审 / 分层定价 |
| 审批 | 门槛微调 / 白名单增补 / 拒绝码复核 |

## 输出模板

```
根因象限：[渠道/产品/定价/审批]
关键指标：获客 <X人/+X%>，激活 <X%/+X%>，[辅助指标]
建议动作：P0-P3 列表
预期恢复时间：[渠道问题 1-2 个月；产品/定价 3-6 个月；审批微调 1 个月内]
```

## 注意事项

- 四象限**不是互斥的**，多因素叠加时按主次列出
- 区域特性影响大（华南客群习惯 vs 华北客群不同），不要用集团平均掩盖区域差异
- 获客指标先看**总量**再看**质量**（激活率、客均贡献），别只看总量
