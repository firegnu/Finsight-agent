# 异常调查标准流程（四步法）

## 何时使用

当某区域某指标偏离历史均值 ≥ 2σ，或连续 2 月环比超 ±15%，触发本流程。

## 四步排查（按顺序，找到根因即停）

### Step 1：审批数据（最常见原因，约 60% 案例归因到这）

检查**异常出现前 3-6 个月**的审批层：
- 信用分门槛是否调整？（最常见：为冲量下调）
- 单月审批通过率是否骤升（> 历史均值 5pp）？
- 新客客群构成：分数分布直方图 vs 历史对比
- 渠道构成变化：低质量渠道占比是否上升？

**判定**：若审批门槛或通过率有异常调整 → 根因确定，进入应对动作。否则继续 Step 2。

### Step 2：催收执行（约 15% 案例）

检查**异常期**的催收执行层：
- 催收团队人力是否充足？（离职率、人均案件数）
- 催收回收率自身是否下滑？
- 外包催收机构变更？
- 催收话术或策略调整？

**判定**：若催收能力下降 → 异常是"滞留未回收"堆积效应，非新增违约。

### Step 3：宏观与客群行为（约 10% 案例）

- 该区域宏观经济：失业率、工资增长率
- 该区域存量客户行为：平均账单金额、循环利息使用率
- 是否有重大事件（政策、自然灾害、舆情）

**判定**：若其他区域同期未见异常、且无本地特异事件，排除宏观因素。

### Step 4：产品/定价（约 15% 案例）

- 近期是否调整了分期利率、服务费？
- 核心产品的权益变更？
- 竞品近期推出更优产品？

## 输出模板

```
根因象限：[审批 / 催收 / 宏观 / 产品]
置信度：[高/中/低]
关键证据：[具体数据点]
建议动作：[P0/P1 行动列表]
验证窗口：[3-6 个月，与审批调整滞后期对应]
```

## 关键注意事项

- **审批调整的逾期影响滞后 3-6 个月**——短期看不到别急着下结论
- 多因素可能叠加——按**贡献度**排序，不是非此即彼
- 报告里要明确"根因推测 vs 已确认"
